# COVA FICC Brief Vision

## 一句话

把分散的公众号 FICC 信息流，变成可持续订阅、可追踪沉淀、可按日交付的专业市场情报服务。

## 我们解决的问题

- 信息源分散，人工跟踪成本高。
- 原文和摘要交付链路不统一。
- 缺少稳定的日报生成机制。
- 生产部署能力不足，难以长期稳定运行。

## 北极星目标

1. 所有目标公众号通过 RSS 方式统一接入。
2. 每篇文章都可追溯（原文可存、摘要可查）。
3. 每天凌晨 4:00 自动产出 FICC 市场简报。
4. 与 COVA 接口兼容，摘要与汇总能力可由 COVA 管理的 LLM 执行。
5. 服务长期在线，具备可配置域名、可观测和可运维能力。

## 阶段目标

### Phase 1（工程化基础）

- 从内存 POC 迁移到持久化后端。
- 打通“采集 -> 入库 -> 摘要 -> RSS 分发”链路。

### Phase 2（日报能力）

- 打通凌晨 4:00 自动汇总。
- 形成标准化日报输出与历史归档。

### Phase 3（平台化对接）

- 完成 COVA 任务接口对接。
- 支持模型治理、重试、审计与成本可视化。

## 成功标准

- 任务稳定按时执行（采集与日报）。
- 原文 RSS 与摘要 RSS 都可被订阅并稳定更新。
- 日报质量可持续迭代，且有明确数据来源边界。
- 部署后可长期运行，故障可监控、可恢复。
